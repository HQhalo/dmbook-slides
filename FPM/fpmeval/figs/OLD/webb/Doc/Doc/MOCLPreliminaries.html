Preliminaries

Magnum Opus analyses data to detect patterns (either rules or itemsets) that capture potentially interesting regularities within those data.

The data is imported from a text file in one of several supported formats (see transaction data and attribute-value data).   Users typically extract data from a database into a text file in a suitable format.  The data import process supports considerable flexibility in the formats that may be employed.

Once the data are imported into Magnum Opus, the system performs a search for patterns.  The user can choose between five measures of the importance for a rule and two measures of importance for an itemset (see search mode).  The user also specifies the maximum number of patterns to be found, and can place further restrictions on patterns to be considered (see commands).

Within the restrictions specified by the user, Magnum Opus finds the patterns with the highest values on the specified measure of importance.  Magnum Opus will only find fewer than the specified number of patterns if the search is terminated by the user or there are fewer than the specified number of patterns that satisfy the user specified constraints.

Holdout evaluation enables potentially spurious patterns to be detected and discarded.

The patterns found are recorded to standard output and may optionally be exported to a comma separated value file suitable for input into a spreadsheet for further analysis.

The tutorial provides an introduction to the system's use.
