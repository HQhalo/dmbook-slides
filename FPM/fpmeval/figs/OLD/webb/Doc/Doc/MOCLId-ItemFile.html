Identifier-item file

See also: transaction data and attribute-value data, item list file, names file, data file.

An identifier-item file is a text file that lists transactions to be analyzed in identifier-item format.  Each transaction has a unique identifier.  Each line associates a transaction identifier with an item in that transaction.

Each line must contain at least two columns, one for the identifier and one for the item.

The following commands are specific to identifier-item files:

identifier-column=[no] Specifies the column that contains the identifier.
item-column=[no] Specifies the column that contains the item.
ordered Specifies that the identifier-item file is ordered by identifier.  Specifying this option can greatly reduce data import time.

For example, the following shows the content of an identifier-item file, called example.idi, containing two columns delimited by colons, the first column containing transaction data, the second the item and the third the identifier.

001, apples
001, oranges
001, bananas
002, apples
002, carrots
002, lettuce
002, tomatoes

Transaction 001 contains apples, oranges and bananas.  Transaction 002 contains apples, carrots, lettuce and tomatoes.  The file is ordered because all data for a transaction appears on consecutive lines of the file.

This data could be processed with the following comand.

mocl id-item-file=example.idi id-col=3 item-col=2 delim=: ordered
