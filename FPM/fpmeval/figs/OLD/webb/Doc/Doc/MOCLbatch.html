Batch Processing

During normal operation Magnum Opus issues progress reports to the terminal.  For long operations, however, it is sometimes desirable to run the system as a background process disconnected from a terminal.  A number of facilities are provided to support such batch processing.

The no-progress-reports command line argument instructs Magnum Opus not to output progress reports.  This is desirable if the output is redirected to a file.

Snapshots can be used to obtain progress reports from a batch process.  A snapshot produces a file containing the current percentage of progress through the search process and the number of rules found so far.

See Snapshots for an example of how to run Magnum Opus as a batch job and take a snapshot during rule discovery.
