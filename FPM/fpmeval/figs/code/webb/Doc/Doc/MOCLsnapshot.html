Taking snapshots while Magnum Opus is running

Magnum Opus includes a unique facility to examine the current best rules while rule discovery is in progress.  When Magnum Opus receives the SIGUSR1 signal it writes the best rules found so far during the search to the file snapshot.out and then resumes the rule discovery process.

Example:

% mocl item-list-file=difficult.itl no-progress-reports > difficult.out &
[1] 16365
% kill -s SIGUSR1 16365
% cat snapshot.out
Rule Discovery Snapshot

Search 11% complete

ST05 -> WA4
[Coverage=0.997 (579415); Support=0.936 (544044); Strength=0.939; Lift=1.00; Leverage=0.0026 (1495.4); p<4.94E-324]

ST25=0 -> WA2=0
[Coverage=0.999 (580538); Support=0.949 (551128); Strength=0.949; Lift=1.00; Leverage=0.0008 (449.6); p<4.94E-324]
