Compute time

Rule discovery by its very nature has very high computation requirements. Magnum Opus uses the computationally efficient OPUS search algorithm. However, even the most efficient search algorithm will be overcome by the requirements of some rule searches that can be devised.

In general the following actions will decrease compute time.

  • Increase the minimum leverage (minimum-leverage option).
  • Increase the minimum coverage (minimum-coverage option).
  • Increase the minimum support (minimum-support option).
  • Decrease the maximum size (maximum-size option).
  • Decrease the maximum number of results to be found (maximum-results option).
  • Decrease the number of values allowed in a pattern (items-available, items-excluded, lhs-available, lhs-exclude, rhs-available and rhs-exclude options).

Note, increasing the minimum lift or strength will only decrease compute time if m-estimates are used (m option) or the minimum coverage or support is set to a high value.

Search by lift and search by strength are both substantially faster if m-estimates (m option) are used.

Magnum Opus stores all data in memory during computation. The single greatest factor that will increase compute time will be when memory requirements exceed available RAM and the system starts utilizing virtual memory. The slow access times for virtual memory greatly increase overall compute time.

The following steps may alleviate compute slowdown caused by utilization of virtual memory.

  • Increase the amount of memory installed in the host computer.
  • Reduce the number of attributes utilized from the data.
  • Reduce the number of attribute values considered during search by using the items-available, items-excluded, lhs-available, lhs-exclude, rhs-available, and rhs-exclude options.
  • Reduce the number of cases to be analyzed, for example, by using the proportion option.

The data format can also greatly affect processing time.  If the data consist of objects that are either present or absent and you are interested only in associations between the presence of objects, then either the item-list or identifier-item file format will result in faster processing than the name file - data file format.  The item-list format is more efficient than the identifier-item format.  Where applicable, using the ordered command line option will greatly speed up processing of identifier-item data.
