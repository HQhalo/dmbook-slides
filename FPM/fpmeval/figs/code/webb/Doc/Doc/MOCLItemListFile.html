Item list file

An item list file is a text file that lists transactions to be analyzed, one to a line.  Each line contains the list of items in a transaction.

For example, the following shows the content of an item list file delimited by commas.

apples, oranges, bananas
apples, carrots, lettuce, tomatoes

This file lists two transactions.  The first transaction contains apples, oranges and bananas.  The second transaction contains apples, carrots, lettuce and tomatoes.

Note: Magnum Opus can only perform very elementary error checking during data import from an item-list file.  In particular, specification of the wrong column delimiter is unlikely to be detected.  For example, if the command line option delimiter=: were specified for the above example, Magnum Opus would interpret the file as containing two transactions each consisting of a single item.
