CSV files

In addition to the standard output, the user can save rules and itemsets in comma separated value format.  This format is suitable for importing into a spread sheet.  Many users find this a convenient way to analyse the associations that are found. 

The File: Export Results menu command in the interactive system and the csv-file option in the command-line system allow the user to select a file into which the associations will be recorded in comma separated value format.  

The first line of this file lists column headers. The remaining lines each describe a single association.  

For rules the following columns are recorded:  

LHS: The left-hand-side of the rule.  

RHS: The right-hand-side of the rule.  

Coverage: The coverage (as a proportion of the data).

Coverage count: The coverage (as a number of cases).  

Support: The support (as a proportion of the data).  

Support count: The support (as a number of cases).  

Strength: The strength.  

Lift: The lift.  

Leverage: The leverage (as a proportion of the data).  

Leverage count: The leverage (as a number of cases).  

If either of the insignificant or unsound filters were used, the p value is also listed in another column.

In addition, if holdout evaluation is performed, depending upon the tests selected, a selection of the following will be included: 

Holdout Coverage: The coverage of the rule on the holdout data (as a count).  

Holdout Support: The support of the rule on the holdout data (as a count).  

Coverage p: The binomial probability of obtaining the observed holdout coverage or higher if the true coverage were the selected minimum coverage.

Support p: The binomial probability of obtaining the observed holdout support or higher if the true support were the selected minimum support. 

Strength p: The binomial probability of obtaining the observed holdout strength or higher if the true strength were the selected minimum strength. 

Lift p: The binomial probability of obtaining the observed holdout lift or higher if the true lift were the selected minimum lift. 

Leverage p: The binomial probability of obtaining the observed holdout leverage or higher if the true leverage were the selected minimum leverage. 

Positive correlation p:  The probability of obtaining the observed support if the true strength of the rule where less than or equal to the frequency of the RHS in the whole dataset.  Calculated using a Fisher Exact Test.  

Improvement with respect to: The subset of the LHSrule with respect to which the rule obtains the largest p for improvement on the holdout data.  

Improvement p: The largest p for improvement on the holdout data.  

Partial with respect to: The superset of the LHSrule with respect to which the rule obtains the largest p for partialness on the holdout data.  

Partial p: The largest p for partialness on the holdout data.  

For itemsets the following columns are recorded:  

Itemset: The itemset.  

Coverage: The coverage (as a proportion of the data). 

Coverage count: The coverage (as a number of cases).  

Leverage: The leverage (as a proportion of the data).  

Leverage count: The leverage (as a number of cases).  

In addition, if holdout evaluation is performed, depending upon the tests selected, a selection of the following will be included: 

Holdout Coverage: The coverage of the rule on the holdout data (as a count).  

Coverage p: The binomial probability of obtaining the observed holdout coverage or higher if the true coverage were the selected minimum coverage. 

Leverage p: The binomial probability of obtaining the observed holdout leverage or higher if the true leverage were the selected minimum leverage. 

Improvement with respect to (a): The first of the pair of subsets of the itemset with respect to which the it obtains the largest p for improvement on the holdout data. 

(b): The second of the pair of subsets of the itemset with respect to which the it obtains the largest p for improvement on the holdout data.  

Improvement p: The largest p for improvement on the holdout data.  

Self-sufficient p: The largest p for self-sufficiency on the holdout data.

Each column is separated by a comma.
